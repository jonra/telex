What it does

📨

Direct & broadcast messaging

Message a session by name, broadcast to all, or talk over named channels — with a blocking wait so exchanges feel synchronous.

🧵

Channels & threads

Group conversations into channels and label threads so several discussions run in parallel without crossing wires.

✅

Task board & harness

Post tasks with dependencies, let workers claim and report them, and drive whole workflows from one config — no per-agent API calls.

📊

Live dashboard & TUI

Watch agents, channels, threads, a message log, an animated network graph, and throughput — pushed live over SSE.

🎛️

Operator console

Drop into a command desk to direct-message individual sessions, post tasks, and steer the whole team at a higher level.

💸

Cost balancing

Tag each session with its model, account, and cost tier, then route work to whichever subscription is cheapest or has quota left.

Quick start

# 1. start the daemon
npm install && node server.js

# 2. connect a session (run in each Claude Code session)
claude mcp add --transport http telex http://127.0.0.1:4123/mcp

# 3. tell the agent
"Call telex_guide, register as 'device', then wait for messages."

Then open the dashboard to watch them talk, or run node tui.js for the terminal view.
